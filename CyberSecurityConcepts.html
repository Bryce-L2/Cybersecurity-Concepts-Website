Cybersecurity Concepts

The CIA Triad

The CIA triad is a way to remember the three main goals of keeping information safe.

Image Source: Logstail.com

Confidentiality

Is the practice of protecting against unauthorized access.

Example: Alice is sending an email to Bob, and an unauthorized user sees the message and posts about it.

Integrity

Is the practice of protecting against unauthorized modification.

Example: Alice sends an email to Bob, and an unauthorized user alters the information in the email. Bob then gets false information.

Availability

The practice of protecting against denial of service.

Example: Alice tries to send Bob an email, but something is blocking the email from being sent to Bob.

Why the CIA Triad Matters

The CIA Triad matters because it is like the three rules of protecting data.

The Three States of Data

We need to know the state of data to then apply the right cybersecurity tool to protect it.

Image Source: Sealpath.com

1. Data at Rest

Definition: Data in storage that is at rest.

Examples: Files on hard drives, databases, backup tapes, cloud storage, archives, etc.

2. Data in Transit

Definition: Data actively moving from one location to another across networks or between devices.

Examples: Email, file transfers, web browsing, video streaming, etc.

3. Data in Use

Definition: Data being actively used, accessed, or changed by applications or users.

Examples: Data loaded in RAM, documents being edited, etc.

Authentication

Authentication is used to prove who you are so that you are given the correct access.

Image Source: Swoopnow.com

Authentication Factors

Something You Know

Examples: Passwords, PINs, etc.

Strengths: Easy to implement and easy for correct user

Weaknesses: Can be forgotten or guessed

Something You Have

Examples: Smart cards, mobile devices, keys, etc.

Strengths: Physical possession required

Weaknesses: Can be lost or stolen

Something You Are

Examples: Fingerprints, facial recognition, eye scans, voice recognition, etc.

Strengths: Unique to individual, can't be forgotten

Weaknesses: Privacy concerns, technological breakdowns or issues

Multi-Factor Authentication

A Combination of two or more authentication factors increases security.

Common MFA Examples:

  • Password + SMS code
  • Fingerprint + PIN
  • Smart card + password
  • Authenticator app + biometric

Password Security

A password is a combination of characters and numbers.

Image Source: Hostpapa.com

Creating Strong Passwords

Think of these steps while creating a password:

  • ✓ Use at least 12-16 characters
  • ✓ Combine uppercase, lowercase, numbers, and special characters
  • ✓ Use unique passwords for each account
  • ✓ Consider using passphrases
  • ✗ Avoid dictionary words, personal information, or common patterns
  • ✗ Never reuse passwords across multiple sites
  • ✗ Don't share passwords via email or text

Password Strength

Weak Password

Example: password

Short, predictable, easily cracked in seconds

Moderate Password

Example: Bryce2009!

Contains some variety but includes personal info and common patterns

Strong Password

Example: kT9$mP2#vL6@qR4

Long, random, includes multiple character types, no patterns

Common Password Attacks

Method #1

  • Shoulder Surfing: Looking over ones shoulder to identify their password.
  • Keylogging: Recording keystrokes to capture passwords.

Method #2

  • Brute Force: Software programs that automate the process of rapidly testing all possible combinations of characters.
  • Uneducated Users: Guessing the password based on knowledge of the user.

What Hackers Know About Passwords

They know:

  • There is a 50% chance that there is a vowel.
  • Capital letters are usually at the beginning and are followed by a vowel.
  • Numbers are usually 1 or 2 and are at the start or end.
  • Women usually tend to use their personal names.
  • Men usually tend to use their hobbies.

Phishing Attacks

Phishing is one of the most successful cyberattack methods, using deception to trick individuals into revealing sensitive information or installing malware.

Image Source: Cyberpilot.io

What is Phishing?

Phishing is the use of bogus emails and websites to trick you into:

  • Revealing passwords, credit card numbers, or other sensitive data
  • Clicking malicious links that install malware
  • Downloading infected attachments
  • Transferring money or making fraudulent payments

Types of Phishing

Spear Phishing

An email scam targeted towards a specific individual, organization, or business.

Whaling

An email scam targeted at high-value individuals.

Smishing

Text messages are used for phishing.

Vishing

Phone calls or voice messages for phishing.

How to Identify Phishing

  • ⚠️ Urgent or threatening language: "Act now or lose everything!"
  • ⚠️ Generic greetings: "Dear Customer" instead of your name
  • ⚠️ Suspicious sender addresses: Slight misspellings or grammatical errors
  • ⚠️ Unexpected attachments: Especially .exe, .zip, or .scr files
  • ⚠️ Requests for sensitive information: Real companies will never ask for passwords via email
  • ⚠️ Mismatched URLs: Hover over links to see actual destination
  • ⚠️ Too good to be true offers: Prize winnings, or free gifts

Protection Strategies

  • Verify before clicking: Contact the organization directly using official contact information
  • Examine URLs: Look for misspellings, extra characters, or wrong domains
  • Keep software updated: Browsers and security software can block known phishing sites
  • Report suspicious emails: Forward to IT department or report to the impersonated organization

Social Engineering

Social engineering is psychological manipulation used on people to perform actions or collect confidential information. It exploits human psychology rather than technical vulnerabilities.

Image Source: Hoxhunt.com

Common Social Engineering Tactics

Baiting

Offering something enticing to lure victims into a trap. Often uses physical media or download links.

Example: USB drive labeled "Winning Lottery #s" found on the ground. Curious person plugs it in, installing malware.

Shoulder Surfing

Looking over someone's shoulder to see their screen, keyboard, or documents to gather information.

Example: Watching someone enter PIN at an ATM or login credentials at an ATM.

Dumpster Diving

Searching through trash to find information that can be used for attacks.

Example: Going behind an office building to look for crumpled papers with potential information.

Info Written in Workspace

Finding passwords or codes that were put up on the walls or laid on a desk.

Example: An interview goes on in an office, revealing emails and passwords of employees written on paper hung on the walls.

Piggybacking

Gaining physical access by following authorized personnel into restricted areas.

Example: Attacker with hands full asks employee to hold door open to secure building.

Pretexting

Creating a fake scenario to obtain information. Attacker poses as someone with authority or legitimate need.

Example: Caller claims to be from IT and needs your password to "fix" your computer.

Ransomware

Using fake or real messages to force users to take an action, like installing fake software or paying a ransom.

Example: Someone claiming to be a teacher at school sends a link. You click it and accidentally download ransomware onto your computer.

Common Cybersecurity Threats

Modern society faces a wide range of cybersecurity threats. Understanding these threats can help create effective defense strategies.

Image Source: Diro.io

Common Threats

Viruses

Programs that attach to legitimate files and spread when those files are executed or shared.

Worms

Programs that replicate and spread across networks without user interaction.

Trojans

Malicious programs disguised as legitimate software that create backdoors for attackers.

Impact: Unauthorized access, data theft, system control

DDoS Attacks

Distributed Denial of Service attacks overwhelm systems with traffic, making services unavailable to legitimate users.

Man-in-the-Middle

Attackers intercept communications between two parties, altering data in transit.

Supply Chain Attacks

Compromising software/hardware in the supply chain to affect multiple downstream targets.

Networking

Learning the key concepts of network are neccessary to understand cybersecurity to its fullest.

What is a Network?

A network is a group of 2 or more computers connected to one another, this gives the user access to the data, resources, and applications.

Types of Networks

LAN

LAN or Local Area Netowrk are devices connected inside a small geographic area for local sharing of resources.

Examples: Home Wifi

WAN

WAN or Wide Area Network are devices connected across a wide geographical area. Usually a collection of connected LANs.

Example: The internet

Network Hardware

Hub

Sends messages to everybody. They just continue moving signals through without determining who the correct reciver is.

Switch

Sends messages only to the destination device in the LAN using MAC addresses. Way smarter than hubs.

Router

Connects different networks together. Called a "gateway" because it's the only device that can connect two networks.

Wireless Access Point

A switch without wires. Allows wireless devices to connect to a wired network.

Network Identifiers

MAC Address

MAC Address or Media Access Control Address is an unique identifier for each device. Represented by a 12 hexadecimal digits and is NOT changeable.

Example: 2C:54:91:88:C9:E3

IP Address

IP Address or Internet Protocol Address is a temporary identifier for each device on a network. Must be unique among connected devices on that network. IS changeable.

IPv4 Example: 192.168.54.96

IPv6 Example: 4353:bd7::4

Important Protocols

TCP or Transmission Control Protocol

Connection oriented and reliable

Keeps track of lost packets and packet order. Uses the 3 way handshake to establish connections.

Used for: Web browsing, email, and file transfers

UDP or User Datagram Protocol

Connectionless and fast

Doesn't track lost packets. Much faster than TCP but less reliable.

Used for: Online gaming, streaming, and DNS

HTTP/HTTPS

Web communication protocols

HTTP is unsecure and unencrypted we browsing

HTTPS is encrypted and secure web browsing

DNS or Domain Name System

Internet's phonebook

Translates human readable domain names into IP addresses.

Example: google.com is 142.250.190.14

DHCP or Dynamic Host Configuration Protocol

IP assignment

Automatically assigns IP addresses, subnet masks, gateways, and DNS servers to devices on a network.

ARP or Address Resolution Protocol

MAC to IP mapping

Tool used by devices to find the MAC address associated with an IP address on the same network.

Network Security Concepts

Subnet Masks and Network Segmentation

A subnet mask is a 32 bit value that separates network bits from host bits. This defines which devices are on the same network.

Example:

  • IP Address: 192.168.1.100
  • Subnet Mask: 255.255.255.0

Ports

Ports are virtual openings that handle specific types of network traffic. Port numbers range from 0-65535.

Common Ports:

  • Port 80 is HTTP
  • Port 443 is HTTPS

Network Analysis Tools

Wireshark

Network packet analyzer that captures and examines network traffic.

Uses: Security analysis and protocol learning

Nmap

Network scanning tool that identifies live hosts, open ports, and running services.

Uses: Security and finding vulnerability
